# 🤝 贡献指南 (Contributing Guide)

感谢您对 **The Architect's Protocol (SOP)** 的关注！本协议的目标是为 AI-Human 协同开发建立一套严谨、可审计、高水准的工程标准。

## 🏛️ 我们的哲学 (Our Philosophy)

在提交 PR 或 Issue 之前，请确保您的建议符合以下核心价值观：
- **原子化 (Atomics)**：任何变更都必须是细粒度的。
- **契约优先 (Contract-First)**：先接口，后实现。
- **人类主导 (Human-in-the-loop)**：AI 是执行者，人类是最终审计员。

## 🛠️ 如何贡献 (How to Contribute)

1.  **提交 Issue**：发现协议漏洞或有新指令提议？请使用我们的 [Issue Templates](file:///Users/jiajiawang/ai-code/.github/ISSUE_TEMPLATE/)。
2.  **提交 Pull Request**：
    - 请基于 `dev` 分支创建功能分支。
    - 确保更新了相应的 [SOP.md](file:///Users/jiajiawang/ai-code/SOP.md) 或 [.cursorrules](file:///Users/jiajiawang/ai-code/.cursorrules)。
    - 在 PR 描述中说明该变更如何提升工程质量。
3.  **代码规范**：
    - 如果是模版更新，请保持 Markdown 格式整洁，使用 GitHub Alerts 强调重点。

## 🌐 语言规范

本协议的所有官方文档、指令名及核心注释**全量使用中文**，以确保架构意图的零偏差对齐。
